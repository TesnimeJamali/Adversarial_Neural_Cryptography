Motivation

Why Neural Cryptography?

🔒 Classical Limits

Static algorithms — fixed by expert design. Non-differentiable, impossible to integrate into gradient-based pipelines. Cannot adapt or evolve.

⚛️ Quantum Threat

Shor's algorithm breaks RSA & ECC. Grover weakens AES-128 to 64-bit. Harvest now, decrypt later — the threat is already here.

🧠 Deep Learning Power

Networks learn vision, language, games from scratch. If they can learn to translate — why not encrypt?

"Cryptography is about making secrets. Machine learning is about finding patterns.
Can we pit them against each other?"
The Central Question

Can neural networks autonomously develop encryption strategies — without being taught any cryptographic primitive — using only adversarial pressure?

Abadi & Andersen · Google Brain · 2016 · arXiv:1610.06918

This is NOT about learning existing ciphers like AES.
It's about discovering entirely new ones from first principles.

The Framework

Three Agents, One Game

Alice
Plaintext P + Key K
Ciphertext C
Eve
C only · No Key ❌
P̂E ❌ Garbled
Bob
Ciphertext C + Key K
P̂B ✅ Correct

Alice

Encrypts P using shared key K. Goal: produce C that Bob can decode but Eve cannot.

Input: 32 bits

Bob

Decrypts C using same key K. Has the structural advantage over Eve.

Input: 32 bits

Eve

Intercepts C only — no key. Structurally disadvantaged: same arch, half the info.

Input: 16 bits
Training Objective

The Adversarial Minimax Game

Alice & Bob's Loss

L_AB = Bob_error
+ (N/2 - Eve_error)² / (N/2)²

Target: Bob → 0, Eve → N/2 = 8

Minimized when Bob decodes perfectly AND Eve is indistinguishable from random guessing (8/16 bits wrong).

Eve's Loss

L_E = Eve_reconstruction_error

Trained separately — 2× steps/round
Independent optimizer

Eve minimises her own error independently. Gets more training steps to stay competitive.

No Dataset — Fully Random

Fresh random (P, K) pairs every step. 20,000 steps × 4,096 batch = 82M+ unique pairs. No labels. No memorization.

Key Design Principles

→ Dense first: global mixing of all bits
→ Conv after: local positional refinement
→ Tanh last: output in (-1,+1) = XOR domain
→ stride=1 everywhere: sequence len stays 16
→ SAME padding: no border information loss

Why {-1,+1} not {0,1}?

XOR in {-1,+1} = multiplication → differentiable. Gradients flow through ciphertext back to Alice. Tanh enforces this domain.

Training Dynamics

The Four Phases of Learning

PHASE 1 · 0–500
🎲 Random
All agents random. Both errors near baseline 8 bits.
PHASE 2 · 500–3k
🔓 Trivial Cipher
Alice/Bob find simple protocol. Eve exploits it immediately.
PHASE 3 · 3k–8k
💥 Crisis
Eve adapts. Bob's error RISES — the adversarial dance.
PHASE 4 · 8k–20k
🔐 Secure
Key-dependent cipher emerges. Eve pushed to random baseline.

Phase 3 rise in Bob's error is a positive sign — adversarial pressure is working.

Non-Monotonic Behaviour

Both too few (1) and too many (4) Eve steps cause failure.
Too few → no adversarial pressure, trivial cipher survives.
Too many → Alice/Bob can't converge in 20k steps.
eve_steps=2 confirms the paper's recommendation.

Learning Paradigm

Why NOT Contrastive Learning?

Contrastive Learning (SimCLR, CLIP...)

→ Pulls similar samples together in embedding space
→ Pushes dissimilar samples apart
→ Goal: learn good representations
→ Needs a labeled/unlabeled dataset
→ One model, no adversarial competition
→ Signal: sample similarity / distance

Adversarial Neural Crypto

→ Three competing agents with conflicting goals
→ Goal: learn to encrypt and decrypt
→ No dataset — randomly generated data
→ Multi-agent adversarial game (GAN-inspired)
→ Signal: agent competition, not sample distance
→ Security emerges from pressure, not similarity
❌ No notion of "similar" ciphertexts — what would that mean?
❌ Security requires a live adversary, not a distance metric
❌ Contrastive learning produces embeddings — not encryption functions
❌ Eve needs to actively attack, not just be a "negative sample"

Both push/pull — but contrastive pushes data points apart · neural crypto pushes agents apart.

Security Analysis

Empirical Robustness vs Provable Security

❌ Not IND-CPA

No proof against all polynomial-time adversaries. Only tested against neural network attackers with the same architecture and bounded training budget.

❌ Not Perfect Secrecy

Key is only 16 bits. Shannon perfect secrecy requires key ≥ message and uniform distribution. Neither condition is satisfied here.

✅ Empirically Robust

5/5 fresh Eves fail to break baseline. Mean error 9.04 bits = 113% of random baseline. All fresh Eves exceeded the random-guess threshold.

Harvest Now, Decrypt Later

Adversaries already collecting encrypted traffic today. Waiting for quantum computers powerful enough to decrypt. The threat is already here — even without quantum computers today.

Neural Crypto's Position

→ No mathematical hardness assumption
→ Shor's algorithm has nothing to factor
→ Security = complexity of learned function
→ Fundamentally different attack surface
→ Not proven quantum-resistant — but different
Neural cryptography doesn't rely on the assumptions quantum computers break. Whether it resists quantum attackers is an open question — and a fascinating one.
Future Work

Open Problems

📏

Scaling to Larger N

16 bits is toy-scale. Can we reach 64 or 128 bits with stable training? Attention may finally prove beneficial at larger N where fixed receptive fields bottleneck.

📐

Formal Security Proofs

Can we connect empirical neural robustness to standard cryptographic assumptions like IND-CPA? This is the central theoretical open problem in the field.

🔗

CBC / CTR Mode Analogues

Current chunking = ECB mode — leaks structure for repeated blocks. Neural equivalents of chaining modes would prevent this pattern leakage.

🔑

Public-Key Neural Crypto

Can we learn asymmetric encryption? This would require a fundamentally different framework — possibly two separate key networks.

👥

Ensemble Adversaries

Would training against multiple simultaneous Eves with different architectures improve robustness? A natural extension of adversarial training theory.

⚛️

Quantum Adversary Testing

Is the learned cipher actually resistant to quantum-accelerated attacks? Connecting neural robustness to post-quantum security definitions.

Conclusion

What This Project Demonstrates

🔐

Neural networks CAN learn to keep secrets

From scratch, without any cryptographic knowledge, using only adversarial pressure — encryption emerges spontaneously.

⚡

The training objective is everything

Loss design, step ratios, architecture details — small changes produce dramatically different outcomes. Quadratic + eve_steps=2 is the sweet spot.

🔬

The gap IS the open problem

We can show it works in practice. We cannot prove it works in theory. Closing this gap would be a major breakthrough.

"Neural networks can learn to keep secrets. But learning to keep them provably — against all possible adversaries — remains an open and profound challenge."
